## O que você vai ver

Esta é uma aula de teste técnico, criada só para validar que o contrato de aula (`lesson.json` + loader) generaliza além da aula `mind-task-flow`.

O assunto é propositalmente simples: o ciclo de vida de um Pull Request — abrir, revisar e fazer merge.

Role pra baixo para começar.

<a id="step-open"></a>
## Dev abre o PR

## 1 — Dev abre o PR

O desenvolvedor termina o trabalho numa branch e abre um Pull Request propondo a mudança para revisão.

<a id="step-review"></a>
## Revisor aprova

## 2 — Revisor revisa e aprova

Um revisor lê o diff, comenta se necessário e aprova quando a mudança está pronta para entrar.

<a id="step-merge"></a>
## Merge na main

## 3 — PR é mesclado na main

Com a aprovação registrada, o PR é mesclado na branch principal — a mudança passa a fazer parte do histórico oficial.

<a id="step-consolidate"></a>
## Ciclo completo

## Ciclo completo

Você percorreu o ciclo mínimo de um Pull Request: abrir, revisar/aprovar e mesclar.

Se isto renderizou corretamente — steps, câmera e diagrama sincronizados — o contrato de aula generalizou sem precisar de nenhum código novo além de trocar o slug carregado.
